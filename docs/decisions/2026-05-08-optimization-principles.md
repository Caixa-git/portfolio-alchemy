# 최적화 원칙: 모듈 단위 측정 + 연구 기반 설계

## 맥락
단순히 기능 구현으로 끝내지 않고, 모든 모듈이 최적화되어야 함.
게임보이 시절의 창의적 최적화 사고(BGM 2배속=효과음 등)를 참고.

## 결정
1. 모든 모듈: **구현 → 성능측정 → 용량측정 → docs/benchmarks/ 기록**
2. 결정은 **논문/연구 기반**으로 (uIP, GCC LTO, Slab Allocator 등)
3. 연구 자료는 `docs/research/`에 정리
4. 4개 분야 우선 참고: 초경량 TCP/IP, 바이너리 최소화, 메모리 최적화, 극한 최적화 사례

## 결과
모든 수치가 문서화되어 개선 추적 가능.
"느낌"이 아닌 "데이터"로 최적화.
