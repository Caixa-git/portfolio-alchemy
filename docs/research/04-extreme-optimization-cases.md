# 극한 용량 최적화 — 창의적 사례

> 게임보이 시절 "BGM 2배속 = 효과음", "구름 팔레트 = 풀" 사고방식

## 게임보이/패미컴 시절

| 사례 | 기법 | 절약 |
|------|------|------|
| 슈퍼 마리오 랜드 | 점프음을 BGM 특정 음의 고속 재생으로 대체 | 효과음 샘플 0KB |
| 포켓몬 레드/블루 | 상태이상 3종을 단일 바이트 XOR 1개로 관리 | 별도 플래그 변수 제거 |
| 배트맨(패미컴) | 모든 스테이지 배경 타일 1세트로 통일, 팔레트만 변경 | 타일 데이터 2KB로 12스테이지 |
| 메가맨 2 | 보스 패턴 루틴 1개로 통일, 변수만 다르게 | 코드 70% 감소 |

## 우리 프로젝트에 적용할 창의적 아이디어

1. **HTTP 헤더를 정적 LUT로** — 모든 요청이 거의 동일한 헤더 사용
2. **JSON 파서를 상태 머신 1개로** — 필드별 파서 분기 대신 단일 상태 머신
3. **소켓 버퍼를 TCP 상태와 공유** — 수신 버퍼가 TCP 연결 구조체의 일부
4. **LLM 응답 청크 처리를 링 버퍼 1개로** — 재조립 버퍼 따로 두지 않음
5. **Discord 게이트웨이 최소 필드만 파싱** — 불필요한 필드는 건너뜀
